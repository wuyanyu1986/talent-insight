# 核心用户 "业务伙伴型HRBP"

林雨萱

HRBP（人力资源业务伙伴），服务于一家中大型科技公司的产品与技术部门

**年龄：** 32岁

**地点：** 上海，中国

> "我希望用数据和洞察赢得业务部门的信任，让他们把我当成真正的战略伙伴，而不只是'帮忙招人的HR'。"
>

## 关于雨萱

林雨萱在HR领域工作8年，前5年负责招聘和培训执行，2年前转型为HRBP。她每天需要同时支持3个业务部门（约200名员工），与部门负责人紧密协作完成人才规划、团队盘点、关键岗位招聘等工作。她深知人才决策对业务成果的影响，但常常因为数据不完整、分析工具缺失而无法提供令人信服的建议，这让她感到自己的专业价值未能充分展现。她渴望从"事务执行者"进阶为"战略业务伙伴"，用科学的人才洞察帮助业务部门达成目标。

## 行为特征

- 工作日程高度碎片化，需要在业务会议、一对一沟通、数据整理等多种场景间快速切换，希望工具能够在短时间内提供准确信息
- 习惯以业务目标为出发点思考人才问题（如"Q3产品上线需要什么样的团队能力配置"），而非单纯关注HR指标
- 在向业务部门提建议前，需要反复验证数据准确性和逻辑严密性，因为一旦判断失误会损害HRBP的专业信誉
- 更倾向于可视化和结构化的信息呈现方式，方便在与业务领导沟通时快速展示关键洞察
- 对新工具的学习成本敏感，希望系统操作直观，不需要长时间培训就能上手使用

## 痛点

- 人才数据散落在HR系统、业务系统、本地Excel、邮件附件等多处，每次做人才分析都需要花费2-3天手动整合数据，效率极低且容易出错
- 业务部门经常提出紧急人才需求（如"下周面试，帮我推荐3个合适的内部候选人"），但她只能凭印象和碎片信息快速拼凑答案，缺乏科学依据让她感到不安
- 无法量化证明人才工作的业务价值，在申请预算或推动培训项目时，很难用数据说服管理层"这笔投入能带来什么回报"
- 当关键人才离职或核心岗位出现空缺时，她才意识到继任规划的缺失，但此时已错过最佳应对时机，只能被动救火
- 业务部门常质疑她的建议"是基于数据还是感觉"，即使她花了大量时间分析，也因无法清晰解释判断逻辑而失去说服力

## 目标

- 快速获得完整可信的人才数据视图，在30分钟内完成原本需要2-3天的数据整合工作，有更多时间用于战略思考和业务沟通
- 用科学的数据分析和AI建议支撑每一个人才决策，让业务部门认可她的专业性，主动寻求她的意见而非仅在招聘时才想起她
- 提前识别人才风险（如关键岗位继任空白、高潜人才流失征兆），有充足时间制定预案，从被动应对转向主动规划
- 清晰量化人才工作的业务贡献（如培训如何提升团队产出、优化人员配置如何改善项目交付率），在管理层面前建立HR职能的战略价值

## 核心任务

- 为业务部门的关键岗位招聘推荐内部候选人，分析候选人与岗位要求的匹配度，提供排序建议和培养计划
- 完成季度或半年度的团队人才盘点，识别团队能力短板、高潜人才和继任风险，输出人才地图和行动建议
- 响应业务部门的临时人才咨询（如"这个项目应该配什么样的团队"、"某员工是否适合晋升"），快速提供基于数据的专业意见
- 设计和推动人才发展项目（如领导力培养、技能提升计划），并追踪项目效果与业务成果的关联
- 监控关键人才指标（如核心岗位继任准备度、团队流动率、能力缺口演变趋势），及时向业务负责人和HR管理层汇报风险与机会
- 在业务战略调整时（如新产品线启动、组织架构优化），评估当前人才储备与新需求的匹配情况,制定人才补充或培养方案
